El estándar diferencia inicialmente tres tipos de etiquetas: Section, grouping content, text-level

1. Sections

Elementos de seccionado, que son los que se reflejan en un esquema de la página

Inicialmente estos elementos incluían body y los encabezados h1...h6

En html se añaden los nuevos elementos semánticos: section, header, article, footer, nav y aside

1.1. Section

Define una sección

Section es un elemento de seccionamiento genérico, diseñado para contener una parte de un documento temáticamente definida. Los contenidos de un elemento section son usualmente precedidos por un encabezado. El elemento section está pensado para agrupar temáticamente un conjunto de elementos. Si existiese la necesidad de definir un bloque con propósitos exclusimavemnte estilísticos o de programación, el elemento div debería utilizarse en su lugar.

1.2. Header

El encabezado de una sección

El elemento header contiene al encabezado de una sección o documento, donde se colocan habitualmente los encabezados, los vínculos de navegación, los formularios de búsqueda, los logos, las tablas de contenidos, las introducciones, etc. En muchos casos, los contenidos del encabezado de un documento son consistentes a lo largo de todo el sitio.

Htmlquick

1.3. Footer

El pie de una sección

El elemento footer representa al pie de una sección o documento, donde los autores habitualmente colocan firmas, información acerca del autor, información de licencias, documentos relacionados, etc. En muchos casos, los contenidos del pie de un documento son consistentes a lo largo de todo el sitio.

Htmlquick

1.4. Nav

Sección de navegación

El elemento nav es un contenedor para enlaces de navegación, que se supone proveen formas de acceder a las secciones del sitio web, así como a documentos externos. Se supone que los vínculos dentro de este elemento corresponden a un bloque de navegación principal únicamente.

Htmlquick

1.5. Article

Contenido redistribuible

El Elemento article de HTML (<article>) representa una composición auto-contenida en un documento, página, una aplicación o en el sitio, que se destina a distribuir de forma independiente o reutilizable, por ejemplo, en la sindicación. Podría ser un mensaje en un foro, un artículo de una revista o un periódico, una entrada de blog, un comentario de un usuario, un widget interactivo o gadget, o cualquier otro elemento independiente del contenido

Htmlquick

1.6. Aside

Apartado secundario

El Elemento HTML Aside (<aside>) representa una sección de una página que consiste en contenido que está tangencialmente relacionado con el contenido que le rodea, que podría ser considerado independiente de ese contenido. Estas secciones son a menudo representadas como barras laterales o como inserciones y contienen una explicación al margen como una definición de glosario, elementos relacionados indirectamente, como publicidad, la biografía del autor, o en aplicaciones web, la información de perfil o enlaces a blogs relacionados. 2. Grouping

Son elementos que no contribuyen o no se reflejan en el esquema de la página

2.1 Párrafo

Inserta un párrafo

El elemento p representa un párrafo. Se considera párrafo a un bloque de texto compuesto de una o más oraciones que se refieren a un tema en particular y que normalmente se separa de otros por un espacio en blanco. Se pueden crear párrafos con las etiquetas, p, pre, blockquote

<P>, se usa para redactar un parrafo relativamente propio, que no pertenezca a una página o un archivo en concreto. <Pre> es un tipo de parrafo prediseñado que guarda la misma apariencia visual, de fuente, tamaño, espacio, tabulaciones y saltos de linea, perfecto para parrafos copiados que queramos que mantengan exactamente la misma composición. y por ultimo <blockquote> es usado unicamente para cubrir uno o varios parrafos sacados de una página, archivo, libro, periodico etc, por lo general acompañado de una etiqueta <cite< con la referencia de donde sacamos el articulo en cuestión.

2.2 Listas

Las listas las podemos diferenciar en tres tipos, lista ordenada (<ol>), lista desordenada (<ul>), lista detalle (<dl>).

El elemento <ol> representa una lista ordenada de ítems que, en contraste con las listas no ordenadas (elemento <ul>), produce un significado diferente si el orden de los ítems es alterado. Los ítems de ambas listas, ordenadas y no ordenadas, son representados por el elemento (<li>).

2.3 div

Contenido genérico para bloques de texto

El elemento div es un contenedor genérico sin un significado semántico en particular. Se utiliza comúnmente en el desarrollo de documentos con propósitos estilísticos, en conjunto con los atributos style y class. También puede resultar útil para proveer atributos comunes a los elementos contenidos por el, como por ejemplo lang o title.

2.4 address

información de contacto del autor

El contenido de un elemento address provee información de contacto para el autor de la sección o documento, como una dirección postal, una dirección de correo electrónico, un número telefónico, una ciudad, un país, un código postal, etc.
